цель:
создать SPA приложение, используя Blazor WASM.
важное замечание, приложение Blazor WASM не должно зависить от сервера (то есть места размещения). это иначе можно сформулировать так - а именно не должно с ним взаимодействовать, то есть по сути может стартовать хоть из папки с файлами. 
для этих целей при создании проекта предлагаю не активировать галку отвечающию за генерацию беккендсервера, то есть будет только проект с клиентом.
если сильно надо можно запускаться и через сервер, но с ним взаимодействовать не надо. то есть его используем просто для запуска.
приложение должно обладать тремя страницами:
1. логин
2. страница один - доступна всем пользователям 
3. страница два - доступна только пользователяи в роли админ ( имя роли не важно)
содерживмое страниц не важно, можно просто что-то выводить хоть данные пользователя, главное чтобы было понятно что это страница из пункта 2 а это страница из пункта 3
важно:
если пользователь не аутентифицирован он всегда попадает на страницу с логином - при этом макет используется без NavMenu - то есть тут свой шаблон
после аутентификации используется основной шаблон страниц с Навигацией по страницам (как в дефолтном шаблоне проекте - с элементами каунтер и.т.д.).
в общем макете страниц (как в дефолтном шаблоне проекта)  есть блок NavMenu, так вот там соотвественно будет два элемента меню (пункты 2 и 3 тз) так вот доступность пункта 3 должна быть реализована через тег AuthorizeView с указанием роли:

минимум:
в качестве факта успешной аутентификации не должен использоваться сервер. то есть мы стартуем приложение  с сервера (можно ибез него) но с сервером больше не взаимодействуем ( в дефолтной реализации BlazorWasm приложения с аутентификацией, она реализована через взаимодействие с сервером и работой с куками или токеном JWT). соотвественно пары логин с ролью и пароль можно защить в коде
